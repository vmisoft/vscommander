# Feature Spec: Make Directory (F7)

> Status: DRAFT
> Source of truth: Far Manager (`../FarManager/far/mkdir.cpp`)
> Generated by the `far-feature-spec` skill on 2026-05-16
> Captures: `specs/make-directory.captures/` (14 states captured from live Far)

## 0. Summary & open questions

- **Effect:** `F7` opens a "Make folder" dialog that creates one or more
  directories in the active panel's directory. It can create intermediate
  (nested) directories, expand environment variables in the name, optionally
  create a directory junction or symbolic link instead of a plain directory,
  and optionally create several directories at once from a delimited list.
- **Trigger:** `F7` while a file panel is focused.
- **Open questions:** the combobox dropdown-list-*expanded* visual did not
  render in captures `09`/`13` (injecting `Space` did not open the list);
  the link-type values are confirmed from source (none/junction/symlink) and
  the closed combobox is captured. Everything else — including conflict,
  filesystem error and archive context — is live-captured (states 11/12/14).

## 4. Focus, keyboard & mouse behaviour

- **Focus cycle (Tab / Shift+Tab):** name edit -> link-type combobox ->
  Target edit (skipped while disabled) -> checkbox -> OK -> Cancel -> wraps
  back to the name edit. Captures `03`-`07` show each focused control; the
  focused control renders with the focus/selected palette (the focus
  highlight) — diff `03-focus-linktype` vs `06-focus-ok` for the colour shift.
- **Combobox (`Up`/`Down`/`Space`):** `DIF_DROPDOWNLIST` — a closed list;
  cycling changes the value among none / junction / symlink;
  `DIF_LISTWRAPMODE` wraps the selection.
- **Enter:** activates `MKDIR_OK` (`DIF_DEFAULTBUTTON`) from any field.
- **Escape:** cancels and closes the dialog (exit code != `MKDIR_OK`).
- **Mouse:** left-click focuses a control / activates a button; the dialog
  body can be dragged. Per-element mouse detail: re-capture with `click`.

## 5. Behaviour, defaults & effect

- **Defaults:** name edit has `DIF_USELASTHISTORY` (history id `NewFolder`) —
  it pre-fills with the **last entered value** (capture `02` shows a stale
  `Soft` from a prior run). Link type defaults to **none**; Target edit starts
  **disabled** (`DIF_DISABLE`). The "Process multiple names" checkbox default
  is the persisted option `Global->Opt->MultiMakeDir` (`mkdir.cpp:146`).
- **Dynamic rule (`MkDirDlgProc`, mkdir.cpp:86-104):** on `DN_EDITCHANGE` of
  the link-type combobox, the Target edit is enabled only when link type is
  not "none" (`DM_ENABLE`).
- **Operation on confirm (mkdir.cpp:158-250):** the name is taken, env vars
  expanded; if "multiple names" is off the whole string is treated as one
  name; if on, it is split on `;`/`,` (quote-aware). For each path, every
  separator yields an intermediate directory — `create_directory` is called
  per segment, skipping segments that already exist (so `a/b/c` creates all
  three). If a link type is selected, `add_reparse_point` makes the last
  directory a junction or symlink to the Target.
- **Post-action:** the created directory is added to the tree; the panel
  refreshes and the cursor moves to the new entry.
- **Persistence:** the checkbox state persists (`Global->Opt->MultiMakeDir`);
  the name/target fields persist via input history.

## 6. Conflict handling (MANDATORY)

`create_directory` is called per path segment; an intermediate segment that
is already a directory is skipped (`os::fs::is_directory(Part)` check,
mkdir.cpp:202), so creating a directory whose name already exists **as a
directory** is a no-op for that segment. Creating one whose name collides
with an existing **file** fails and routes to the error dialog in section 7.
VERIFIED — capture `12-error` (mkdir of `existingfile.txt`).

## 7. Filesystem error handling (MANDATORY)

`create_directory` wraps the OS call in `retryable_ui_operation(...)` with
`lng::MCannotCreateFolder` (mkdir.cpp:106-109); link creation uses
`lng::MCopyCannotCreateLink`. A failure raises Far's retryable error dialog.
VERIFIED from capture `12-error` — the dialog is:

- **Title:** "Cannot create the folder"
- **Body:** the full target path, then the OS error, e.g.
  `0x000000B7 - Cannot create a file when that file already exists.`
- **Buttons:** `{ Retry } [ Skip ] [ Skip all ] [ Cancel ]` — four buttons
  (note: more than the Retry/Skip/Cancel a source read alone would suggest;
  `Skip all` sets the `SkipErrors`/`SkipAll` flag that suppresses repeats).

The same dialog covers permission denied, path not found, invalid name,
read-only filesystem, etc. — only the OS error line changes. An empty/invalid
multi-name list instead raises a separate `MWarning` / `MIncorrectDirList`
message with a single **OK** (mkdir.cpp:227-233).

## 8. Archive context (MANDATORY)

VERIFIED — capture `14-archive-mkdir`. When the active panel is inside an
archive (top border shows `ArcLite:zip:<file>`), `F7` is routed through the
archive plugin, not `ShellMakeDir`. It opens a **minimal single-field
dialog** titled with just an **"Enter directory name:"** input — no link-type
combobox, no Target, no "process multiple names" checkbox. Read-only archive
formats (TAR/RAR) reject the operation with an error.

## 9. F1 help (MANDATORY)

`F1` in the dialog opens Far's help at topic `@MakeFolder` (set by
`Dlg->SetHelp(L"MakeFolder")`, mkdir.cpp:150). Capture `08-f1-help` shows the
help window. The topic explains: environment-variable expansion in the name,
creating nested subfolders by separating names with the path separator, the
Link Type option, and the "Process multiple names" delimited-list behaviour.
VSCommander documents the equivalent in `docs/MakeFolder.md`.

## 11. Midnight Commander cross-reference

MC's "Make directory" (`mc/src/filemanager/cmd.c`) is a single-line input
dialog; it also supports nested paths but has no link-type option and no
multi-name list. Far governs; the link-type and multi-name features are
Far-specific and part of this spec.
