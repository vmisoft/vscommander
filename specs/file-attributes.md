# Feature Spec: File Attributes (Ctrl+A)

> Status: DRAFT
> Source of truth: Far Manager (`../FarManager/far/setattr.cpp`)
> Generated by the `far-feature-spec` skill on 2026-05-16
> Captures: `specs/file-attributes.captures/`

## 0. Summary & open questions

- **Effect:** `Ctrl+A` opens a dialog to view and change the attributes of the
  selected file(s) — in VSCommander, **platform-adaptive**: on Linux/macOS/
  FreeBSD it edits POSIX permissions, owner/group, and timestamps; on Windows
  it would edit NTFS attributes (Windows variant is out of scope for v1 — see
  section 12).
- **Trigger:** `Ctrl+A` while a file panel is focused.
- **Open questions:** `chown` to a *different* owner needs elevated
  privileges; tests run as a normal user, so owner/group changes are tested
  only as same-value no-ops / dialog display, not privileged re-owning.

## 2. Activation & trigger

- **Trigger:** `Ctrl+A`, panel focused. Acts on the **selected files** (or the
  cursor entry when nothing is selected) — consistent with Copy/Move.
- **Key table:**

| Key | Context | Action |
|---|---|---|
| `Ctrl+A` | Panel focused | Open the File attributes dialog |
| `Tab` / `Shift+Tab` | Dialog | Cycle focus (wraps) |
| `Space` | Checkbox focused | Toggle |
| `Enter` | Dialog | Apply (the **Set** default button) |
| `Esc` | Dialog | Cancel, no change |

## 3. UI specification

### Far (Windows) reference — capture `01-dialog`

Far's `Ctrl+A` dialog (`setattr.cpp`) is a Windows NTFS dialog: ~10 attribute
checkboxes (Read only / Archive / Hidden / System / Compressed / Encrypted /
Not indexed / Sparse / Temporary / Offline) plus a `[ More ]` advanced
sub-dialog, the four NTFS timestamps (write / creation / access / change) with
`[ Original ] [ Current ] [ Blank ]` buttons, the Owner, and
`{ Set } [ System properties ] [ Cancel ]`. See
`specs/file-attributes.captures/01-dialog.ansi.txt` / `.png`.

### VSCommander Unix dialog (this spec's design)

A new platform-adaptive dialog — see section 12 "Intentional deviations".
Built with `formView` (like `mkdirPopup`). Controls, top to bottom:

| Group | Control |
|---|---|
| Header | label — `Change attributes for: <name>` or `<N> selected items` |
| Permissions | interactive `rwx` checkbox grid (Owner/Group/Other x Read/Write/Exec) + Special row (setuid/setgid/sticky) + a live octal read-out — custom control `PermGrid` |
| Owner / Group | name field; an editable combo (`NameCombo`) with a `[v]` dropdown of names from `/etc/passwd`/`/etc/group` when available, else a plain input — a numeric id is also accepted |
| Modified / Accessed | opt-in masked field (`MaskedDate`): a gating checkbox enables a digits-only `YYYY-MM-DD hh:mm:ss` mask; an impossible date shows `(invalid)` and blocks Set |
| Recurse | checkbox — `Apply changes to directory contents` (only when a directory is targeted) |
| Buttons | `{ Set }` (default) `[ Cancel ]` |

The three controls are custom interactive controls in `src/attrControls.ts`;
they required a small `formView.ts` extension so `addCustom` controls can
receive keyboard input. The approved mockup is
`specs/file-attributes.captures/proposed-dialog.png`.

Colours: the standard VSCommander dialog theme (border / text / edit / button /
checkbox), reusing existing theme elements — no new theme element required.

## 4. Focus, keyboard & mouse behaviour

- **Focus cycle:** the 12 permission checkboxes -> Owner -> Group -> Modified
  -> Accessed -> Recurse -> Set -> Cancel -> wraps. The focused control draws
  with the dialog focus/selected palette.
- **Space:** toggles the focused checkbox; toggling a permission checkbox
  updates the octal readout.
- **Enter:** applies (`Set`).
- **Escape:** cancels with no filesystem change.
- **Mouse:** left-click focuses/toggles a control or activates a button;
  the dialog body is draggable (standard popup behaviour).

## 5. Behaviour, defaults & effect

- **On open:** read the cursor/first-selected entry's current `stat` — mode
  bits, owner, group, mtime, atime — and pre-fill the dialog with them. When
  multiple files are selected with differing values, a differing field starts
  blank/indeterminate.
- **On Set:** for every target, apply the changed fields — `chmod` for the
  mode, `chown` for owner/group (only if changed), `utimes` for the
  timestamps (only if changed). Unchanged fields are left untouched.
- **Recurse:** when the recurse checkbox is on, a target that is a directory
  has the changes applied to its whole subtree.
- **Post-action:** both panes refresh.
- **Persistence:** none — the dialog always reflects live filesystem state.

## 6. Conflict handling

Attributes do not "conflict" the way a copy target does. If a file is changed
on disk between opening the dialog and pressing Set, the new values simply
overwrite — no merge. N/A beyond that.

## 7. Filesystem error handling (MANDATORY)

Applying an attribute can fail: permission denied (not the owner; `chmod`/
`chown`/`utimes` on a file you do not own), path no longer exists, read-only
filesystem, `chown` without privilege. Each failure raises VSCommander's
standard retryable error dialog — **Retry / Skip / Cancel** — reusing the
`extension.ts` error-callback pattern already used by mkdir and delete; a
`SkipAll`-style flag suppresses repeats across a multi-file / recursive run.

## 8. Archive context

N/A — attributes cannot be changed for entries inside an archive. When the
active panel is inside an archive, `Ctrl+A` is a no-op (Far disables the
dialog there). VSCommander: ignore the key in archive mode.

## 9. F1 help (MANDATORY)

Far's dialog help topic is `@FileAttrDlg`. VSCommander documents the feature
in a new `docs/Attributes.md` (and the `USER.md` section), reachable from the
F1 help index `docs/Contents.md`.

## 10. Edge cases

| # | Edge case | Expected behaviour | Status |
|---|---|---|---|
| 1 | No selection | Acts on the cursor entry | VERIFIED (design) |
| 2 | Multiple selected, differing modes | Differing field starts indeterminate | ASSUMED |
| 3 | Recurse on a directory | Whole subtree updated | design |
| 4 | Symlink under cursor | `chmod`/`utimes` act on the link target (POSIX default); document this | ASSUMED |
| 5 | `chmod` on a file not owned | Permission-denied error dialog | design (section 7) |
| 6 | `chown` to a different user without privilege | Error dialog; tests use same-owner no-op | VERIFIED (constraint) |
| 7 | Invalid timestamp text | Field rejected / dialog stays open with the field flagged | design |
| 8 | `..` entry under cursor | `Ctrl+A` ignored (cannot attribute the parent ref) | design |
| 9 | Inside an archive | `Ctrl+A` ignored (section 8) | design |

## 11. Midnight Commander cross-reference

MC splits this into separate `chmod` and `chown`/`advanced chown` dialogs
(`mc/src/filemanager/chmod.c`, `chown.c`, `achown.c`). MC's chmod dialog is a
toggleable list of the permission bits — a good UX reference for the Unix
permissions section. VSCommander combines permissions + owner + timestamps
into one dialog (closer to Far's single-dialog model).

- **Constraint:** `attrPopup.ts`, `fileOps.ts` must not import `vscode`.
- **Existing implementation:** none — VSCommander has no attribute feature
  today. This is a new feature, not a delta.
- **New theme elements:** none — reuse the dialog theme.

### Intentional deviations from Far

| Deviation | Far behaviour | VSCommander behaviour | Reason |
|---|---|---|---|
| Platform-adaptive dialog | Windows NTFS attributes only | Unix variant edits POSIX permissions / owner / group / timestamps; Windows variant (NTFS) deferred | VSCommander is cross-platform (Linux/FreeBSD/macOS/Windows); Far's NTFS dialog is meaningless on POSIX filesystems |
| Single combined dialog on Unix | (Far is Windows-only) | One dialog covers permissions + owner + timestamps | Fewer dialogs than MC's separate chmod/chown; matches Far's single-dialog feel |

v1 ships the **Unix variant**; the Windows NTFS variant is a documented
follow-up.
